# Step 1 — r2e-core: Foundations

## Goal

Implement the framework core: `AppState`, `AppBuilder`, error handling, and the `Controller` trait that will serve as the base for macro-generated code.

## Files to Create

```
r2e-core/src/

  state.rs        # Generic AppState
  builder.rs      # AppBuilder
  error.rs        # Error types + IntoResponse
  controller.rs   # Controller trait
```

## 1. AppState (`state.rs`)

The AppState is a generic container wrapping application services.

```rust
use std::sync::Arc;

/// Application state container.
/// T is a user-defined struct containing their services.
#[derive(Clone)]
pub struct AppState<T: Clone + Send + Sync + 'static> {
    inner: Arc<T>,
}

impl<T: Clone + Send + Sync + 'static> AppState<T> {
    pub fn new(inner: T) -> Self {
        Self { inner: Arc::new(inner) }
    }

    pub fn get(&self) -> &T {
        &self.inner
    }
}
```

### Why Generic?

Each application defines its own services. `AppState<T>` allows maintaining strong typing without resorting to `TypeId` or `Any`.

## 2. AppBuilder (`builder.rs`)

Fluent builder for constructing the Axum application.

```rust
pub struct AppBuilder<T: Clone + Send + Sync + 'static> {
    state: Option<T>,
    routes: Vec<axum::Router<AppState<T>>>,
}

impl<T: Clone + Send + Sync + 'static> AppBuilder<T> {
    pub fn new() -> Self { ... }

    /// Sets the application state
    pub fn with_state(mut self, state: T) -> Self { ... }

    /// Adds a router (generated by a controller)
    pub fn register_routes(mut self, router: axum::Router<AppState<T>>) -> Self { ... }

    /// Builds the final Router
    pub fn build(self) -> axum::Router { ... }

    /// Builds and starts the server
    pub async fn serve(self, addr: &str) -> Result<(), Box<dyn std::error::Error>> { ... }
}
```

### Behavior of `build()`

1. Merges all registered routers
2. Applies `.with_state(AppState::new(state))`
3. Returns an `axum::Router` ready to serve

## 3. Error Handling (`error.rs`)

```rust
// Note: these types are re-exported via r2e::prelude::*
use r2e_core::http::response::{IntoResponse, Response};
use r2e_core::http::header::StatusCode;

/// Standard application error
pub enum HttpError {
    NotFound(String),
    Unauthorized(String),
    Forbidden(String),
    BadRequest(String),
    Internal(String),
}

impl IntoResponse for HttpError {
    fn into_response(self) -> Response {
        let (status, message) = match self {
            HttpError::NotFound(msg)     => (StatusCode::NOT_FOUND, msg),
            HttpError::Unauthorized(msg) => (StatusCode::UNAUTHORIZED, msg),
            HttpError::Forbidden(msg)    => (StatusCode::FORBIDDEN, msg),
            HttpError::BadRequest(msg)   => (StatusCode::BAD_REQUEST, msg),
            HttpError::Internal(msg)     => (StatusCode::INTERNAL_SERVER_ERROR, msg),
        };

        let body = serde_json::json!({ "error": message });
        (status, axum::Json(body)).into_response()
    }
}
```

Also provide conversions like `From<sqlx::Error>`, `From<std::io::Error>`, etc. to enable the use of `?` in handlers.

## 4. Controller Trait (`controller.rs`)

```rust
/// Trait implemented by generated controllers.
/// Provides the method to register routes.
pub trait Controller<T: Clone + Send + Sync + 'static> {
    /// Returns the Router containing all routes for this controller.
    fn routes() -> axum::Router<AppState<T>>;
}
```

This trait will be automatically implemented by the `#[controller]` macro.
